# Plan prompts

Specs for Claude Code (or other agents) **`/plan`** sessions. These are **task prompts**, not ADRs — outcomes land in `docs/architecture/adr-*.md` after review.

**Naming:** `YYYY-MM-DD-{milestone-slug}-plan-prompt.md`. Add a new file and table row for each plan pass; do not overwrite generic names or reuse an executed prompt as a living doc.

| File | Purpose |
|------|---------|
| [2026-07-22-post-v030-adr-backlog-plan-prompt.md](2026-07-22-post-v030-adr-backlog-plan-prompt.md) | Post–v0.3 architecture ADR backlog (Proposed ADRs 017–024) |

Session outputs (optional, not committed by default) can go in `sessions/` with dated filenames, e.g. `sessions/2026-07-22-post-v030-adr-backlog-plan.md`.
